General data object

The DataObject is an abstract class, which is intended to simplify common set/get functionality of data objects. Every protected member of the child class is considered a property of the data object, unless the member name is in the ignore array, set by DataObject::registerIgnoreProps().

For example: the class

  1.  class Test extends DataObject
  2.  {
  3.      protected $testValue;
  4.      protected $testNoProperty;
  5.  
  6.      public function __construct()
  7.      {
  8.          $this->registerIgnoreProps(array('testNoProperty'));
  9.          parent::__construct();
  10.      }
  11.  }

will provide the following methods:

  1.  $test = new Test();
  2.  $test->setTestValue('test');
  3.  if ($test->hasTestValue()) {
  4.      echo $test->getTestValue();
  5.  }
  6.  $test->test_value = 'test';
  7.  echo $test->test_value;

Note, that the direct access method uses the underscore-separated form. You can take advantage of that to instantiate objects through database backends, where you usually want the underscore-form for table columns.

Also note, that you will have to call the parent constructor too. But after setting the ignore properties.

Interal get/set methods are named register/retrieve to avoid collision with properties.
